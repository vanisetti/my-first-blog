Programming 4 - Blog Post Project
Goal
I wanted to create a website which allowed college students to share and donate used textbooks in order to help people save money and encourage people to continue their studies. In order to create the website, I learned how to create a blog using a tutorial from Django Girls. This tutorial taught me the basics of creating a completely functioning blog.
Data
For my project, I didn't really rely much on data. I used the sqlite3 database for my project because it is the default Django database adapter. Django was the website which provided tutorials and I used the one catered to creating a blog. They also made me install Django which provided me with a framework to develop websites.

I knew how everything was connected and how it affected each other. Here are some things I didn't know (much)about before this project- Virtual Environment: It is an isolated working copy of Python which allows you to work on a specific project without worry of affecting other projects. It enables multiple side-by-side installations of Python, one for each project. It doesnâ€™t actually install separate copies of Python, but it does provide a clever way to keep different project environments isolated. PythonAnywhere: Deploying a web app on PythonAnywhere involves pulling down your code from GitHub, and then configuring PythonAnywhere to recognise it and start serving it as a web application. There are manual ways of doing it, but PythonAnywhere provides a helper tool that will do it all for you Models: A model in Django is a special kind of object which is saved in the database. Models in the database is like a spreadsheet with columns (fields) and rows(data). Views: A view is a place where we put the "logic" of our application. It will request information from the model you created before and pass it to a template. QuerySets: A QuerySet is, in essence, a list of objects of a given Model. QuerySets allow you to read the data from the database, filter it and order it. QuerySets also allow you to order the list of objects. You can also combine QuerySets by chaining them together which is really powerful and lets you write quite complex queries.
