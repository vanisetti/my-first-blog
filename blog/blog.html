Programming 4- Blog Post Project
I originally wanted to create a website and I even had a name planned out for it: Shelf It. “Shelf It” is a website/app which would act as a platform for people to donate books to others. It would consist of many genres of books ranging from textbooks to picture books. It would also target people of all ages but high school and college students in particular. College textbook prices are incredibly high, and once the student graduates, their textbooks become useless. Instead of letting the textbooks languish in a dumpster, “Shelf It” ensures that the books can be put to good use to those in need. Whoever receives these books can use them for however long they are needed, and can later donate them on the website to pass on the knowledge. This cycle of sharing books could enable students to obtain a great education, allow people to feed their curiosity, and enable people to discover their passion for reading without having to worry about the cost. Before checking out, people can choose to pay for the book(s) they want, with the proceeds going to a charity in their area. “Shelf It” will spread love for books, save the environment and help the community grow.
In order to have this website make a difference, I had to learn how to make a website. So I began by learning how to create a blog. A blog has many of the same aspects I was looking for in my website. It has an admin page, I learn how to enter data and have it show up in the format I want. Hence, it was a great first step to learning the basics of creating a website. I used a website called Django Girls (https://tutorial.djangogirls.org/en/) which had a tutorial on how to create a blog. This tutorial also had in-depth explanations of how everything worked which was really helpful because I not only learned how to create a website but also many of the basics about python.
PROCESS
Command Line → Terminal Code Editor → Sublime Text Server → PythonAnywhere With regards to the code, I used sublime text and the terminal. I installed the 3.6 version of python for my project. There are a lot of server providers available on the internet, but I used PythonAnywhere. PythonAnywhere is free for small applications that don't have too many visitors so it was very convenient for me. I also had to install Django (version 1.11.0). Django is a free and open source web application framework, written in Python. It consists of components that help develop websites faster. Django is very useful especially when requesting things from webserver. When a request comes to a web server, it’s passed to Django which tries to figure out what is requested. This is done by Django’s urlresolver. Django checks patterns to see if something matches and then it passed the request to the associated function--which is called the view. In the view function you can look for information in a database. The view checks if you are allowed to do something and generates a response which is used by Django to send to the user’s web browser.
